⚙ How Online Play Works

GlassChess uses Firebase Realtime Database as the backend — it's free, instant, and requires zero server setup. Follow these steps to enable live online play:

  1. Go to console.firebase.google.com and sign in with a Google account.
  2. Click "Add project" → name it (e.g. glasschess) → click through the setup steps.
  3. In the left sidebar, click "Build" → "Realtime Database" → click "Create database". Choose any region. Select "Start in test mode".
  4. In the left sidebar, click the ⚙ gear icon → "Project settings" → scroll down to "Your apps" → click the </> Web button → register your app → copy the config object shown.
  5. Open this HTML file in a text editor. Find the firebaseConfig block at the very top and replace each "YOUR_..." value with your actual values.
  6. Save the file. Host it on any free static host: GitHub Pages, Netlify, or Vercel. Both players open the same URL.
  7. One player clicks "Play Online" → "Create Room". They share the 6-letter room code with their friend. The friend clicks "Play Online" → "Join Room" and enters the code.

🏗 How the Online Code Works

Once Firebase is connected, here's exactly what happens when two players play online:

// ── CREATE ROOM ────────────────────────────────── // Host generates a 6-char room code and writes initial state: set(ref(DB, `rooms/${code}`), { board: serializeBoard(), // 64-sq array → JSON turn: "w", moves: [], white: hostName, black: null, // guest fills this on join status: "waiting" }); // ── JOIN ROOM ───────────────────────────────────── // Guest writes their name and triggers game start: update(ref(DB, `rooms/${code}`), { black: guestName, status: "playing" }); // ── SYNC MOVES ──────────────────────────────────── // After every move, active player writes the new state: update(ref(DB, `rooms/${code}`), { board: serializeBoard(), turn: nextTurn, lastMove: { from, to, san } }); // ── LISTEN (both players) ───────────────────────── // onValue fires whenever ANY field in the room changes: onValue(ref(DB, `rooms/${code}`), snapshot => { const data = snapshot.val(); if (data.lastMove && data.lastMove !== myLastMove) { applyOpponentMove(data.lastMove); // update board } }); // ── CHAT ────────────────────────────────────────── update(ref(DB, `rooms/${code}/chat`), { [Date.now()]: { name: myName, msg: chatMsg } });

🔒 Firebase Security Rules

In your Firebase console, go to Realtime Database → Rules and paste these rules. They allow read/write for 24 hours (good for testing). For production, add proper auth.

// Firebase Realtime Database Rules { "rules": { "rooms": { "$roomId": { ".read": "true", ".write": "true" } } } }
